## v0.5.0 — SPU 五态、类目树、共享包（当前基线，2026-06-03）

### SPU 商品状态（Breaking：枚举扩展）

| 状态值 `SpuStatus` | 中文   | C 端列表 | 可购买       |
| ------------------ | ------ | -------- | ------------ |
| `NOT_LISTED`       | 未上架 | 否       | 否           |
| `ON_SALE`          | 已上架 | 是       | 是（有库存） |
| `SOLD_OUT`         | 已售罄 | 是       | 否           |
| `OFF_SALE`         | 已下架 | 否       | 否           |
| `RESTOCKING`       | 补货中 | 是       | 否           |

- 原 `DRAFT` 废弃，等同 `NOT_LISTED`；seed 会自动迁移历史数据。
- 库存变更后自动同步：已上架 ↔ 已售罄；补货中有库存 → 已上架。
- 实现位置：`packages/shared/src/spu-status.ts`，文档见《共享接口与约定》。

### 管理后台

- 商品列表：**状态列**、状态筛选、行内切换。
- **类目管理**：由扁平表格改为**树形表格**，支持「新增一级类目」「新增子类目」。
- `@simplemall/shared`：开发态直连源码，避免 Vite 预构建缓存缺少新导出（如 `SPU_STATUS_LABEL`）。

### Web 商城

- 商品列表/详情展示售罄、补货中标签；不可购时禁用加购并提示。

### 文档同步

## v0.10.0 — TCB 全量部署（2026-06-04～05）

### 背景

首期将 API、Web、Admin 部署至腾讯云开发：云托管跑 NestJS + Prisma + 云开发 MySQL；静态托管同域名承载商城与管理端。上线后发现 Web 在根路径 `/`、Admin 在 `/admin` 时，Admin 路由或 401 会误跳 `/login`（商城登录页），两端 token 混用；后续在同版本内补齐子路径隔离与 Admin 客户端路由修复。

### 交付内容

| 模块 | 内容 |
| ---- | ---- |
| 数据访问 | **沿用 Prisma + `DATABASE_URL`**，对接云开发 MySQL（控制台 ORM → Prisma 指引），**不**引入 `@cloudbase/node-sdk` `rdb()` |
| 云托管 | `deploy/tcb/Dockerfile` 多阶段构建、`deploy/tcb/env.example`；`prisma generate` 引擎打入镜像，修复云托管未初始化崩溃 |
| API 运行 | `HOST=0.0.0.0`、`PUBLIC_API_BASE` 上传绝对 URL、`ENABLE_CRON` 多副本开关；`PrismaService` 在 `DATABASE_URL` 无效时跳过 `$connect`，保证 `/api/v1/health` 可探活 |
| 静态路径 | Web → `/web`（`NUXT_APP_BASE_URL=/web/`）；Admin → `/admin`（`VITE_BASE=/admin/`）；根 `/` 跳转 `/web/` |
| 部署脚本 | `build-static.sh`、`deploy-static.sh`（`pnpm run deploy:tcb:static`）、`cleanup-legacy-web.sh`、`hosting-routing-rules.example.json` |
| 管理端 | `BrowserRouter` `basename`；`handleAdminUnauthorized` 跳转 `/admin/login`，不再误跳商城 `/login` |
| 工程 | `cloudbaserc.json`、根及 `deploy/tcb` `.dockerignore`（排除 `**/.env`） |
| 文档 | `docs/TCB部署说明.md`、`.env.tcb.example` |

### 涉及模块

`deploy/tcb`、`cloudbaserc.json`、`apps/api`（main、upload、app.module、nest-cli、prisma.service）、`apps/admin`（vite base、basename、401 跳转）、`package.json`（根）、`Dockerfile`、`.dockerignore`、`docs`

### 升级注意

**云托管 API**

1. 云开发控制台创建 MySQL，复制 **内网** 连接串到 `DATABASE_URL`，另配 `REDIS_URL` 等（见 `deploy/tcb/env.example`）后重新部署。
2. 配置数据库后执行 `prisma migrate deploy`。
3. 根目录 `.env` 配置 `CLOUDBASE_ENV_ID`（勿提交），`tcb framework deploy` 或 `pnpm run build:tcb:api`。
4. CLI 灰度提示可回答 `No` 或使用 `printf 'n\n' | tcb cloudrun deploy ... --force`。

**静态托管 Web / Admin**

5. 设置 `NUXT_PUBLIC_API_BASE`、`VITE_API_BASE`、`VITE_API_V2_BASE` 后执行 `pnpm run deploy:tcb:static`（上传 `/web`、`/admin`、根 `index.html`）。
6. 从旧根路径 Web 迁移后运行 `cleanup-legacy-web.sh --yes`，删除根下 `login/`、`products/`、`_nuxt/` 等遗留。
7. 配置 SPA 回退（`/web/*` → `web/200.html`，`/admin/*` → `admin/index.html`）；`CORS_ORIGINS` 同域填一个静态域名即可。

**其他**

8. 容器内 `uploads/` 非持久化，生产图片建议后续接 COS。
9. 本地开发未设 `NUXT_APP_BASE_URL` / `VITE_BASE` 时行为不变。
