## v0.3.0 — 三端核心业务闭环（2026-06-03）

### API（`apps/api`）

- **用户端**：注册/登录、类目树、SPU 列表/详情、购物车、地址、下单、模拟支付、订单列表/详情/取消/确认收货/退货申请。
- **管理端**：类目 CRUD、SPU/SKU 矩阵创建与编辑、上下架、SKU 库存调整、订单列表/详情/发货/退货审核、SUPER 模拟支付回调。
- **领域**：库存预占与释放、订单超时取消、已发货自动完成（定时任务）、`UserTypeGuard` / `AdminTypeGuard`。

### Web 商城（`apps/web`）

- 首页、分类、商品列表/详情（SKU 选择）、购物车、结算、订单、用户地址、登录/注册。

### 管理后台（`apps/admin`）

- 登录、仪表盘、类目、商品、订单、模拟支付（SUPER）。

### 数据

- Prisma `seed`：管理员 `admin/admin123`、用户 `13800138000/user123`、演示 SPU/SKU。

## v0.4.0 — 演示数据、工程修复与后台体验（2026-06-03）

### 数据与脚本

- **Seed 扩充**：9 个类目（含二级）、12 个 SPU、24 个 SKU，覆盖服装/数码/家居/食品。
- `dev.sh` 启动流程增加 `prisma seed`；修复 `ROOT` 未绑定变量等问题。
- 管理端登录 401：JWT 配置、`seed` 与 HTTP 错误提示优化。

### Web（Nuxt）开发环境

- macOS `vite-node` Unix socket 路径过长：Web 使用 `TMPDIR=/tmp`（`apps/web/scripts/dev.sh`）。
- `#app-manifest` 预转换失败：关闭 `experimental.appManifest`，避免误删 `.nuxt` 的 `predev`。
- `dev.sh` 修复 `exec TMPDIR=...` 导致启动失败的问题。

### 管理后台 UI（v0.4）

- 深色侧栏、面包屑、统一页头与卡片布局。
- 登录页分栏品牌区；仪表盘统计与快捷入口。
- 商品列表缩略图、价格格式化、订单状态彩色标签。

## v0.5.0 — SPU 五态、类目树、共享包（当前基线，2026-06-03）

### SPU 商品状态（Breaking：枚举扩展）

| 状态值 `SpuStatus` | 中文   | C 端列表 | 可购买       |
| ------------------ | ------ | -------- | ------------ |
| `NOT_LISTED`       | 未上架 | 否       | 否           |
| `ON_SALE`          | 已上架 | 是       | 是（有库存） |
| `SOLD_OUT`         | 已售罄 | 是       | 否           |
| `OFF_SALE`         | 已下架 | 否       | 否           |
| `RESTOCKING`       | 补货中 | 是       | 否           |

- 原 `DRAFT` 废弃，等同 `NOT_LISTED`；seed 会自动迁移历史数据。
- 库存变更后自动同步：已上架 ↔ 已售罄；补货中有库存 → 已上架。
- 实现位置：`packages/shared/src/spu-status.ts`，文档见《共享接口与约定》。

### 管理后台

- 商品列表：**状态列**、状态筛选、行内切换。
- **类目管理**：由扁平表格改为**树形表格**，支持「新增一级类目」「新增子类目」。
- `@simplemall/shared`：开发态直连源码，避免 Vite 预构建缓存缺少新导出（如 `SPU_STATUS_LABEL`）。

### Web 商城

- 商品列表/详情展示售罄、补货中标签；不可购时禁用加购并提示。

### 文档同步

- 架构说明书 v1.1、共享约定、三端开发设计文档、`docs/README.md` 变更规则均已更新。

### 升级注意

```bash
pnpm --filter @simplemall/shared build
cd apps/api && pnpm prisma:seed
./dev.sh stop && ./dev.sh
# Admin 若报 shared 导出错误：
cd apps/admin && rm -rf node_modules/.vite && pnpm dev
```
