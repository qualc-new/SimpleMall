# 共享接口与约定

> 三端（API / Nuxt / React Admin）共同遵守的契约。  
> 类型定义落地于 `packages/shared`，各端通过 workspace 引用。

---

## 1. 基础约定

| 项           | 约定                                                   |
| ------------ | ------------------------------------------------------ |
| Base URL     | 开发 `http://localhost:4000/api/v1`；生产由 Nginx 反代 |
| Content-Type | `application/json`                                     |
| 鉴权头       | `Authorization: Bearer <accessToken>`                  |
| 管理端前缀   | 业务接口路径以 `/admin` 开头（除登录外）               |
| 金额         | **分**（integer），展示时 `/100` 保留两位小数          |
| 时间         | ISO 8601 字符串（UTC 存库，前端按本地展示）            |
| 分页         | Query: `page=1&pageSize=20`；响应见 §2.3               |

## 2. 统一响应格式

### 2.1 成功

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "req_abc123"
}
```

列表：

```json
{
  "code": 0,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

### 2.2 业务错误

```json
{
  "code": 40001,
  "message": "库存不足",
  "requestId": "req_abc123"
}
```

HTTP 状态码：4xx/5xx 与 `code` 并存；前端以 `code` 做分支。

### 2.3 常用业务错误码

| code  | 含义         | 典型场景                               |
| ----- | ------------ | -------------------------------------- |
| 0     | 成功         | —                                      |
| 40100 | 未登录       | Token 缺失或过期                       |
| 40300 | 无权限       | 管理端 RBAC                            |
| 40400 | 资源不存在   | 订单/商品已删                          |
| 40901 | 订单状态非法 | 状态机边不允许                         |
| 40902 | 库存不足     | 下单/加购超限                          |
| 40903 | 重复支付通知 | 回调幂等（服务端吞掉，仍返回 success） |
| 42200 | 参数校验失败 | DTO 校验                               |

**展示文案映射**（各端可复用同一 map）：

| OrderStatus | 中文   |
| ----------- | ------ |
| PENDING_PAY | 待支付 |
| PAID        | 已支付 |
| SHIPPED     | 已发货 |
| COMPLETED   | 已完成 |
| CANCELLED   | 已取消 |
| REFUNDING   | 退货中 |
| REFUNDED    | 已退货 |

| SpuStatus   | 中文   | C 端列表 | C 端可购 |
| ----------- | ------ | -------- | -------- |
| NOT_LISTED  | 未上架 | 否       | 否       |
| ON_SALE     | 已上架 | 是       | 是（有库存） |
| SOLD_OUT    | 已售罄 | 是       | 否       |
| OFF_SALE    | 已下架 | 否       | 否       |
| RESTOCKING  | 补货中 | 是       | 否       |

**状态同步规则**（`packages/shared/src/spu-status.ts`）：

- 管理端将商品设为 **已上架** 且全部 SKU 可用库存为 0 → 自动变为 **已售罄**。
- **已售罄** 且补货后可用库存 &gt; 0 → 自动恢复 **已上架**。
- **补货中** 且可用库存 &gt; 0 → 自动变为 **已上架**。
- **未上架**、**已下架** 不随库存自动变更，仅后台手动修改。
- 调整 SKU 库存（`PATCH /admin/skus/:id/stock`）后触发 SPU 状态同步。

### 4.2 SPU 列表项 / 详情

SPU 表（`spus`）对齐传统 `goods` 主表字段；**实际售价**以 SKU `price`（分）为准，SPU 层存市场价/成本价/会员价等参考价。

| goods 字段 | SimpleMall `spus` | 说明 |
| ---------- | ----------------- | ---- |
| goods_sn | goodsSn | 唯一货号，可空自动生成 |
| goods_name | title | 商品名称 |
| short_name | shortName | 短标题 |
| subtitle | subtitle | 卖点 |
| brand_id | brandId | 关联 `brands` |
| cate1/2/3_id | cate1Id, cate2Id, cate3Id | 保存类目链，叶子为 categoryId |
| category_path | categoryPath | 如 `1,2` |
| main_img | mainImage | 主图 |
| slide_imgs | imagesJson | 轮播 JSON 数组 |
| spec_type | specType | 0 单规格 1 多规格 |
| attr_json | attrJson | 商品参数 JSON |
| tag_list | tagList | 逗号分隔标签 |
| market_price / cost_price / vip_price | marketPrice / costPrice / vipPrice | **分** |
| sale_price | SKU.price | 在 SKU 矩阵维护 |
| total_stock / sale_num | totalStock / saleNum | 总库存自动汇总；销量暂 seed/预留 |
| warn_stock | warnStock | 预警值 |
| express_id / freight_type | expressId / freightType | 运费 |
| limit_buy | limitBuy | 限购 |
| is_new / is_hot / is_recommend | isNew / isHot / isRecommend | 营销标 |
| status | status | SpuStatus 字符串枚举 |
| sort | sort | 排序 |
| putaway_time | putawayTime | 上架时间 |

## 6. Token 约定

| Token             | 有效期（建议） | 存储                                                    |
| ----------------- | -------------- | ------------------------------------------------------- |
| accessToken       | 2h             | C 端：Cookie `httpOnly` 或 localStorage（开发可用后者） |
| refreshToken      | 7d             | httpOnly Cookie                                         |
| admin accessToken | 8h             | Admin：sessionStorage                                   |

刷新：收到 `40100` 时调用 `/auth/refresh` 一次，失败跳转登录。

---

## 7. 模拟支付签名（开发）

```
X-Mock-Sign = HMAC_SHA256(MOCK_PAY_SIGN_SECRET, rawBody)
```

回调 Body 需包含 `paymentNo`、`orderNo`、`amount`、`channel`。
