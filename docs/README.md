# SimpleMall 文档索引

## 是否需要多端开发设计文档？

**结论：需要。**

| 判断依据         | 说明                                                                                         |
| ---------------- | -------------------------------------------------------------------------------------------- |
| 技术栈异构       | 服务端 NestJS、商城 Nuxt 3、后台 React，架构说明书无法覆盖各端目录、路由、状态管理与 UI 规范 |
| 可并行实施       | P0～P4 分阶段中，三端需独立排期；缺端级文档会导致接口字段、枚举、错误码理解不一致            |
| 架构说明书定位   | 《轻量电商系统架构说明书》= **What & Why**（领域、状态机、库存、部署）                       |
| 开发设计文档定位 | 各端 + 共享约定 = **How**（页面清单、模块划分、请求封装、表单与权限）                        |

**不需要单独拆文档的部分**：Docker、纯运维脚本、压测报告 — 仍写在架构说明书第 12～13 节即可。

## 文档体系

```
docs/
├── README.md                          ← 本索引
├── 版本迭代说明.md                     ← 发版记录、升级注意（推荐阅读）
├── 轻量电商系统架构说明书.md           ← 系统级（必读）
├── 共享接口与约定.md                   ← 三端共同契约（必读）
├── 服务端开发设计文档.md               ← apps/api
├── Web商城开发设计文档.md              ← apps/web (Nuxt)
└── 管理后台开发设计文档.md             ← apps/admin (React)
```

### 阅读顺序（新成员）

1. [轻量电商系统架构说明书](./轻量电商系统架构说明书.md)
2. [共享接口与约定](./共享接口与约定.md)
3. [版本迭代说明](./版本迭代说明.md)（了解已交付功能与升级步骤）
4. 按岗位阅读对应端开发设计文档

### 变更同步规则

| 变更类型                     | 需更新的文档                            |
| ---------------------------- | --------------------------------------- |
| 订单状态、库存策略、支付流程 | 架构说明书 + 共享约定 + 受影响端文档    |
| 新增/修改 API 路径或 DTO     | 共享约定 + 服务端 + 调用方（Web/Admin） |
| SPU 商品状态枚举或同步规则   | 共享约定 + 架构说明书 + 三端 + 版本迭代说明 |
| 发版、里程碑、升级注意       | 版本迭代说明 + 根 README 必要时          |
| Agent 默认行为（命名、记版本） | `.cursor/rules/` + 根目录 `AGENTS.md`    |
| 仅 UI/交互调整               | 对应端开发设计文档                      |
